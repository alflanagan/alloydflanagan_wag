This is a **Figma-ready component list** tailored specifically for a **consulting-focused developer website**. It’s organized so you can map each item directly to Frames, Components, and Variants in Figma.

---

## 1. Global / Foundation

### Design Tokens
- **Color styles**
  - Primary (brand / accent)
  - Neutral 900–50 (text → background)
  - Success / Subtle accent (optional)
- **Text styles**
  - H1 – Hero headline
  - H2 – Section headers
  - H3 – Card titles
  - Body – Primary
  - Body – Muted
  - Small / Caption
- **Spacing scale**
  - 4, 8, 12, 16, 24, 32, 48, 64
- **Radius styles**
  - Small (buttons, tags)
  - Medium (cards)
- **Shadow styles**
  - Card / Elevated

---

## 2. Navigation

### Header / Navbar
**Component:** `Navbar`
- Logo / Name (text-based preferred)
- Nav links
  - Services
  - Work
  - About
  - Contact
- CTA button variant
  - “Book a Call”

**Variants**
- Default
- Scrolled (with background + shadow)
- Mobile (hamburger)

---

## 3. Hero (Consulting-Oriented)

### Hero Section
**Component:** `Hero`
- Headline (problem + outcome driven)
- Supporting paragraph
- Primary CTA button
  - “Schedule a Consultation”
- Secondary CTA (text link)
  - “View Case Studies”
- Optional trust indicator
  - Logos or short credibility line

**Variants**
- With illustration
- Text-only

---

## 4. Trust & Credibility

### Client Logos / Social Proof
**Component:** `ClientLogoRow`
- 4–6 monochrome logos
- Optional caption text

### Testimonials
**Component:** `TestimonialCard`
- Quote
- Name
- Role / Company

**Variants**
- Single featured
- Carousel
- Grid

---

## 5. Services

### Services Section
**Component:** `ServiceCard`
- Icon (simple line icon)
- Service title
- 1–2 sentence outcome-focused description
- Optional bullet list

**Common services**
- Architecture & System Design
- Performance Optimization
- API & Backend Development
- Technical Leadership / Advisory

**Variants**
- With pricing hint (“Starting at…”)
- With CTA (“Learn more”)

---

## 6. Case Studies (Critical for Consulting)

### Case Study Card
**Component:** `CaseStudyCard`
- Client / Project name
- Problem summary
- Outcome / metric
- Tech stack tags
- CTA: “Read case study”

**Variants**
- Featured (larger)
- Standard

---

## 7. Process (How You Work)

### Process Steps
**Component:** `ProcessStep`
- Step number or icon
- Step title
- Short explanation

**Typical steps**
1. Discovery
2. Proposal
3. Execution
4. Delivery & Support

**Layout variants**
- Horizontal (desktop)
- Vertical (mobile)

---

## 8. About / Authority

### About Summary
**Component:** `AboutBlock`
- Short bio (consultant-focused)
- Years of experience
- Industries served

### Credentials
**Component:** `CredentialList`
- Certifications
- Notable companies
- Open source highlights

---

## 9. Call to Action (High Priority)

### Primary CTA Block
**Component:** `CTABlock`
- Headline
- Supporting text
- Primary button
  - “Book a Free Intro Call”
- Secondary text
  - Email link

**Variants**
- Light background
- Accent background

---

## 10. Contact

### Contact Section
**Component:** `ContactCard`
- Email address
- Calendly / booking link
- Location (optional, city only)

### Contact Form (optional)
**Component:** `ContactForm`
- Name
- Email
- Message
- Submit button

---

## 11. Footer

### Footer
**Component:** `Footer`
- Navigation links
- Social links
- Copyright

## Recommended Figma Structure

- Pages
  - Foundations
  - Components
  - Sections
  - Page – Home
- Use Auto Layout everywhere
- Create button + card variants early
- Lock spacing + typography before visual polish
